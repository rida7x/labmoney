# مختبر الهيثم للتحاليل الطبية — نظام الإدارة المالية

نظام إدارة مالية متكامل ومتجاوب للمختبرات الطبية، مصمم خصيصًا للسوق الليبي بالعملة الليبية (د.ل) ودعم كامل للغة العربية واتجاه RTL.

---

## ✨ المميزات

- **لوحة تحكم تفاعلية** مع رسوم بيانية مباشرة (إيرادات، مصروفات، صافي الربح، طرق الدفع).
- **إدارة الإيرادات** بطرق دفع متعددة: كاش، بطاقة مصرفية، يسر باي، موبي كاش، مصرفي باي.
- **إدارة المصروفات** بتصنيفات قابلة للتخصيص.
- **الديون والموردين** مع شريط تقدم لنسبة السداد، سجل الدفعات، وتنبيهات المتأخرات.
- **Lab to Lab** لإدارة العينات الخارجية وأسعار التحاليل.
- **الموظفين والرواتب** (شهري ويومي) + نظام السلف + كشوف رواتب جاهزة للطباعة.
- **التقارير المتقدمة**: يومية، شهرية، سنوية، أو فترة مخصصة، مع تصدير CSV وطباعة PDF.
- **إعدادات شاملة**: طرق الدفع، تصنيفات المصروفات، المستخدمين والصلاحيات، إعدادات المختبر، النسخ الاحتياطي.
- **الوضع الليلي والنهاري** مع حفظ التفضيل.
- **PWA قابل للتثبيت** على iPhone، Android، والكمبيوتر — يعمل دون اتصال بعد التثبيت.
- **نسخ احتياطي واستعادة** بصيغة JSON.
- **متعدد المستخدمين** بصلاحيات: مدير عام، محاسب، عارض فقط.
- **طباعة الفواتير** مباشرة من المتصفح.
- **تصميم متجاوب 100%** للهاتف والكمبيوتر اللوحي وسطح المكتب.

## 🚀 التشغيل المحلي

النظام يعمل بأي خادم ملفات ثابت. لا يحتاج إلى Node.js أو قاعدة بيانات خارجية.

### الطريقة الأولى: Python (الأبسط)
```bash
cd alhaytham-lab
python3 -m http.server 8000
```
ثم افتح المتصفح على: <http://localhost:8000>

### الطريقة الثانية: Node.js
```bash
cd alhaytham-lab
npx serve .
```

### الطريقة الثالثة: PHP
```bash
cd alhaytham-lab
php -S localhost:8000
```

> ⚠️ **مهم**: لا تفتح `index.html` بنقر مزدوج (file://) — استخدم خادم ملفات لأن Service Worker و IndexedDB يحتاجان http://.

## 📱 التثبيت على iPhone (PWA)

1. افتح الموقع في **Safari** (يجب أن يكون Safari وليس Chrome).
2. اضغط زر **المشاركة** (المربع مع السهم).
3. اختر **إضافة إلى الشاشة الرئيسية**.
4. سيظهر التطبيق كأيقونة مستقلة وستفتح بدون شريط العنوان كتطبيق أصلي.

نفس الخطوات على Android (مع Chrome): القائمة ⋮ → "تثبيت التطبيق".

## 💾 قاعدة البيانات

النظام يستخدم **IndexedDB** للتخزين المحلي في المتصفح. هذا يعني:
- ✅ يعمل بدون إنترنت.
- ✅ سرعة فائقة.
- ✅ لا تحتاج إلى خادم.
- ⚠️ البيانات محلية على نفس المتصفح فقط — احرص على **النسخ الاحتياطي الدوري** من قسم الإعدادات.

### النسخ الاحتياطي

1. **الإعدادات → النسخ الاحتياطي → تصدير نسخة احتياطية** → سيتم تنزيل ملف `.json`.
2. احفظه في مكان آمن (Google Drive, Dropbox, إلخ).
3. للاستعادة: **استيراد نسخة احتياطية** → اختر الملف.

## ☁️ الترقية إلى قاعدة بيانات سحابية (اختياري)

لمزامنة البيانات بين عدة أجهزة، يمكنك ترقية النظام إلى Firebase أو Supabase:

### Firebase Firestore
1. أنشئ مشروعًا على <https://console.firebase.google.com>.
2. أضف تطبيق Web وانسخ الـ config.
3. عدّل ملف `js/db.js` — استبدل دوال `add/update/get/getAll/delete` بنداءات Firestore:
   ```javascript
   // مثال
   async add(storeName, data) {
     await firebase.firestore().collection(storeName).doc(data.id).set(data);
   }
   ```
4. أضف Firebase SDK في `index.html`.

### Supabase
نفس الفكرة باستخدام `@supabase/supabase-js` — استبدل طبقة `db.js` فقط دون تعديل بقية الكود.

> الكود مصمم بحيث **قاعدة البيانات معزولة في `db.js`** — كل بقية الملفات تستدعي `db.add`, `db.get`, إلخ، لذا الترقية تعديل ملف واحد فقط.
